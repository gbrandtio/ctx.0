# AGENTS.md — Acme

This workspace was generated by ctx.0. It has two sides that share one wire protocol.

## Routing

| Task involves… | Go to |
|---|---|
| Flutter, Dart, widgets, Bloc/Cubit state, mobile UI | `app/` (see `app/AGENTS.md`) |
| .NET, C#, ASP.NET Core, EF Core, PostgreSQL, migrations | `api/` (see `api/AGENTS.md`) |
| Enabling/disabling features | run `ctx0 enable/disable <id>` — never hand-edit generated wiring |
| The security plane | `app/lib/security/`, `api/src/Security/` — vendored; change deliberately |

## Rules

1. **Do not weaken the security plane.** ALE, request signing, JWT, RLS, envelope
   encryption and rate limiting are load-bearing. `ctx0 doctor` verifies integrity.
2. **Secrets come from environment variables only.** Never commit real keys.
3. **The two sides implement one protocol** (`X-Ctx-Protocol`). A change to signing
   or ALE must update both sides and the shared vectors together.
4. **Feature wiring is engine-owned.** Add business logic freely; leave
   `// ctx:anchor:*` / `# ctx:anchor:*` markers intact.
