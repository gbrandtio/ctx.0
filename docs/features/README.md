# Feature Specs

This directory holds the functional specs of **your product's features** — the business logic you insert on top of the template's shipped capabilities.

- `FEATURE_SPEC_TEMPLATE.md` — copy it to `<FEATURE>.md` for every business feature **before** implementing it. Agents read the spec first.
- `AUTHENTICATION.md`, `LOGIN.md`, `SIGNUP.md`, `USER_PROFILE.md` — pre-structured specs for the flows every app ships; fill in the product-specific slots.

Both templates' `AGENTS.md` route agents here. A feature without a spec here is not ready to implement.
